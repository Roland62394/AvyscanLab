                                  1.  OVERVIEW
=============================================================================================

AvyScan Lab allows you to fix common defects (grain, dust, stabilisation, etc.) and adjust levels and colours to bring your films back to life.
Its role is to write .avs scripts and hand their execution over to the AviSynth frameserver, which must be installed on the workstation.
AviSynth must also have the required DLLs in its x64 directory for its filters to work.
AvyScan is aimed at both hobbyists looking for a turnkey solution and advanced users wishing to customise their restoration pipeline.
Its modular design and user-friendly interface offer great flexibility to adapt to all types of projects and skill levels.
AvyScan Lab lets you choose and individually configure each filter you want to apply.
Its full modularity allows you to add your own custom filters and share them with the community.
Batch encoding with FFmpeg gives you complete flexibility to create high-quality masters or intermediate files for further work.


Standard mode:
        • easy installation with no special knowledge or need to find external components
        • check for a newer version availability
        • application available in 4 languages (FR, EN, DE, ES)
        • user guide in 4 languages as a mini word processor editable by the user
        • import one or more videos or image sequences by simple drag and drop
        • manage multiple clips of different formats in the same project
        • choose the filters you want to apply and in what order (crop, degrain, denoise, sharpen, stabilisation, etc.)
        • adjust the parameters of each filter in a user-friendly interface
        • each parameter of each filter is explained with a dedicated tooltip to understand its purpose
        • compare the original image and the restored image
        • view the histogram for precise control of exposure and colours (4 modes)
        • create a settings library (presets) to recall them in 2 clicks
        • import and export filters from a library or with other users
        • encoding of videos or image sequences to intermediate or final formats with complete flexibility for each clip
        • create an encoding parameters library (presets) to recall them in 2 clicks and apply them individually to each clip


Expert mode:
        • edit the generated AviSynth script and paste its content into another application
        • write your own filters directly in the application, import your DLLs as well as your .avsi files
        • configurable and automatic generation of controls used to modify filter parameters from the interface
        • AviSynth errors are displayed in the application for quick diagnosis
        • freely change the order of filters in your script
        • duplicate filters to apply them multiple times in your script while keeping the ability to adjust/modify each duplicate
        • import and export filters with other users
        • all filters (including your own) can be recalled in 2 clicks and applied individually to each clip

And many more possibilities to discover as you use the application.

                           2.  PREREQUISITES AND INSTALLATION
=============================================================================================

System requirements:
    • Windows 10 or 11 (64-bit)
    • 4 GB RAM minimum (8 GB recommended)
    • Sufficient disk space for source files and renders

Components:
    • AviSynth+ 64-bit must be installed. If absent, AvyScanLab detects it at startup and offers to install it.
    • FFmpeg is bundled with the application (Plugins/ffmpeg/ folder).
    • The mpv player is bundled with the application (mpv/ folder).

No additional installation is required. Simply extract the archive and launch the AvyScanLab.exe executable.

              • A slider or field allows you to set parameter values.

=============================================================================================
                            4.  SOURCE SELECTION
=============================================================================================

Video file
    Drag and drop a video file onto the playback area, or click the button
    next to the "source" field to open a file browser.
    Supported files: .avi, .mp4, .mov, .mkv, .wmv, .mpeg, .mpg.

Image sequence
    Select a directory containing image files (for example: 00001.tif, 00002.tif, etc.).
    AvyScanLab automatically detects the first and last image in the sequence.
    Supported formats: .tif  .tiff  .jpg  .jpeg  .png  .bmp

Frame rate (FPS)
    The "fps" field sets the playback frame rate.

Source loader
    The drop-down menu forces the decoding plugin:
        Auto      Automatic detection (recommended)
        FFMS2     FFmpegSource2 — maximum compatibility
        L-SMASH   LSmashSource — optimised MP4/MOV
        LWLibav   LWLibavVideoSource — alternative format
    If a reading error occurs when opening a file, try changing the source loader.

Threads
    Number of AviSynth processing threads (depends on your processor). Increase this value to use
    all processor cores.
    A value that is too high may cause slowdowns or errors on less powerful machines.
    Adjust it according to your hardware (ideally between 4 and 8).

Multi-clips
    You can load multiple files in AvyScan Lab. Each clip will have its own filter settings
    and its own preset. Drag and drop multiple files at once
    or use the + button to add more.


=============================================================================================
                         5.  VIDEO PLAYER AND NAVIGATION
=============================================================================================

The built-in player displays the AviSynth script result in real time.
Any parameter change automatically regenerates the script and refreshes the preview.
While changes are being calculated, the "Play" button turns orange and changes brightness.
When the program is ready, this button turns green again.

Keyboard shortcuts:

    Key                       Action
    -----------------------------------------------------------------------------------------
    Space                     Play / Pause
    Left arrow                Previous frame
    Right arrow               Next frame
    Ctrl + Left               Go to beginning
    Ctrl + Right              Go to end
    F11                       Maximise/Restore player size

Parameter interactions:

    Action                    Effect
    -----------------------------------------------------------------------------------------
    Click on slider           Positions the cursor directly
    Drag the slider           Real-time adjustment
    Scroll wheel on text field  Adjusts value by step
    Direct input              Type a value, confirm with Enter or click elsewhere
    Hover over a label        Displays tooltip with range and tips for the selected parameter

Half-resolution mode:
    The "1/2" button activates half-resolution mode: the AviSynth script processes the image at half its size BEFORE the heavy filters, then displays it.
    This considerably speeds up preview on less powerful machines.

Histogram:
    The 📊 button in the transport bar shows a visual scope next to the image to
    analyse color and luminance distribution. Useful for checking exposure, detecting
    clipping and tuning white balance.

        Left-click     Toggle the scope overlay on / off.
        Right-click    Open a menu to pick the scope type.

    Available scope types:
        Histogram           Classic histogram: pixel density per brightness level (single plot,
                            luma + chroma overlaid). Best for spotting clipping in shadows and
                            highlights.
        Histogram Parade    Three stacked histograms (R, G, B). Ideal for white balance and
                            detecting color casts.
        Waveform            Luma/chroma waveform: the horizontal axis matches the X position in
                            the image, the vertical axis is the level. Very useful for finding
                            local imbalances (e.g. a blown-out edge).
        Vectorscope         Shows hue and saturation as a point cloud inside a chroma circle.
                            Ideal for judging color fidelity and detecting hue drift.

    The scope adapts automatically to half-resolution mode and to Preview mode (rendered side).

Error display:
    If an AviSynth script error occurs, a banner appears in the player and an error window opens with:
        • The error message (without FFmpeg technical prefixes)
        • The script excerpt around the faulty line (with marker ►)
        • The complete FFmpeg output in a collapsible "Details" section
        • A "Copy" button to copy the complete message

Preset per clip:
    The drop-down menu next to the transport bar allows you to apply a filter preset only to the active clip.


=============================================================================================
                              6.  FILTER PANEL
=============================================================================================

The lower panel is divided into two columns separated by a resizable splitter
(drag it horizontally to adjust the width).


    LEFT COLUMN: FILTER LIST
    -----------------------------------------------------------------------------------------

    Header
        The "FILTERS" title is accompanied by three buttons:
            ↲   Reset filter order to their default position.
            ↓   Import a custom filter from a .json file.
            +   Create a new custom filter (opens the editing form).

    Each filter is represented by a row with three elements:

        ON/OFF button       Click on the filter name to activate (green) or deactivate (grey) it.
                            A deactivated filter is not applied to the script but remains in the list.

        Expand button       The arrow (>) opens the filter parameter panel in the right column.
                            It becomes (v) when the panel is open.
                            A second click closes the panel.
                            This button is hidden for filters that have no adjustable parameters.

        Edit button         The pencil opens the full editing form for the filter (modify code, duplicate, export or delete the filter).


    Reordering filters
        Hold-click on a filter row and drag it up or down to change its position in the pipeline.
        A green horizontal line indicates the insertion point. Release to confirm the new position.
        The displayed order corresponds to the application order in the AviSynth script.

    Inactive filters
        Deactivated filters are grouped in a collapsible section at the bottom of the list.
        Click on the "Inactive filters (n)" header to show or hide this section.
        Drag-and-drop reordering also works within this section.

    General options (outside filters) are:
        preview            AviSynth preview (before/after comparison)
        enable_flip_h      Horizontal flip
        enable_flip_v      Vertical flip

    Processing pipeline order:
        1. Flip
        2. Crop
        3. Dust removal (denoise / RemoveDirt)
        4. Grain reduction (degrain / MVTools2)
        5. Brightness levels (luma levels)
        6. Colour correction (GamMac)
        7. Custom filters (6 possible injection points)
        8. Sharpening (sharpen) — always last


    RIGHT COLUMN: FILTER PARAMETERS
    -----------------------------------------------------------------------------------------

    When you click on the arrow (>) of a filter, its parameter panel is displayed in the right column.
    Multiple panels can be open simultaneously and are displayed side by side horizontally.

    If no panel is open, the text "← select a filter" is displayed.

    Each panel contains:

        Title bar           The filter name in uppercase. You can drag this bar horizontally to reorder panels
                            among themselves (a green vertical line indicates the insertion point).

        Reset button (↺)    Resets all filter parameters to their default values,
                            without closing the panel.

    Parameter control types:

        Slider              A horizontal slider with a text input on the right.
                            Move the slider or enter a value manually.
                            Values are automatically snapped to the step defined for each parameter.
                            Use the mouse scroll wheel on the slider to adjust the value by increments.

        Drop-down list      A list of predefined choices. Select the desired value from the menu.
        (combo)

        Checkbox            An ON/OFF toggle button. The values associated with each state are defined by the filter
                            (for example "true"/"false" or "enabled"/"disabled").

        Text field          A free text input field. The value is saved when you leave the field (click elsewhere or Tab).


    Each parameter has a tooltip (hover over its name) that describes its role and impact.
    Names with a tooltip are underlined.

    Any parameter change immediately updates the video preview, allowing you to adjust settings in real time.


    REGION DRAWING (DRAW REGION)
    -----------------------------------------------------------------------------------------

    Some filters (like Crop) offer a "Draw Region" button in their parameter panel.
    This feature allows you to visually define an area on the image instead of manually entering coordinates.

    Usage:
        1. Open the filter parameter panel (button >).
        2. Click the "Draw Region" button (it turns green when active).
           Playback is automatically paused.
        3. On the video image, hold the left mouse button and draw a rectangle.
           An overlay frame indicates the selected area in real time.
        4. Release the mouse: the coordinates are automatically transferred
           to the filter parameters and the preview is updated.
        5. Drawing mode is automatically deactivated after the drawing. You can
           also deactivate it manually by clicking the button again.

    Two region modes exist depending on the filter:
        • XYWH mode: defines the position (X, Y) and size (Width, Height)
          of the selected area.
        • Crop mode: defines the cropping margins (left, top, right, bottom)
          from the drawn rectangle.

    Coordinates are automatically converted to source pixels (original resolution),
    regardless of the display mode (preview, half-resolution).


    Note: only one filter can be in drawing mode at a time.
           Activating drawing on another filter automatically deactivates the previous one.


    CUSTOM FILTER MANAGEMENT
    -----------------------------------------------------------------------------------------

    Add ("+"):
        Opens the creation form.
        Fill in the name, pipeline position, AviSynth code and parameter controls.
        The filter is added in an activated state.

    Import ("↓"):
        Opens a file selector (defaults to the Filters/ folder).
        A .json file can contain one or more filters.
        Each imported filter receives a new identifier to avoid conflicts.

    Edit ("✎"):
        On the filter row, opens the editing form with the filter pre-filled.

    Duplicate:
        From the editing form, creates a copy of the filter placed right after the original.
        Useful for applying the same processing with different settings.

    Export:
        From the editing form, saves the filter to a .json file that you can share.

    Delete:
        From the editing form, permanently deletes the filter and all its settings.

    Reset ("↲")
        In the header, restores the default order of built-in filters.
        User-created filters retain their current position.

    Slider values are automatically snapped to the step defined
    for each parameter, which prevents invalid values.

=============================================================================================
                         8.  RESTORATION FILTERS
=============================================================================================

    DEGRAIN — Grain reduction (MVTools2)
    -----------------------------------------------------------------------------------------
    Temporal grain reduction via MVTools2 motion analysis.
    The filter compares multiple consecutive frames to distinguish detail from grain.

    Parameter              Range                        Default     Description
    degrain_mode           mdegrain1/2/3/temporal       mdegrain2   Algorithm
    degrain_thSAD          0-1000                       350         Luma threshold
    degrain_thSADC         0-1000                       250         Chroma threshold
    degrain_blksize        4/8/16/32/64                 16          Block size
    degrain_overlap        0 to blksize/2               blksize/2   Overlap
    degrain_pel            1 or 2                       2           Sub-pixel precision
    degrain_search         0-5                          2           Search mode
    degrain_prefilter      remgrain/blur/none           remgrain    Prefilter

    Automatic constraints:
        • blksize can only take values 4, 8, 16, 32 or 64 (powers of 2, MVTools2 MAnalyse requirement).
          The slider adjusts automatically.
        • overlap is automatically capped at blksize/2. If you reduce blksize, overlap is reduced accordingly.
        • overlap only accepts even values (step = 2).

    Notes:
        • mdegrain2: 2 temporal references (recommended, good quality/speed ratio)
        • mdegrain3: 3 references (better result, slower)
        • mdegrain1: 1 reference (fast, less effective)
        • temporal : TemporalSoften (without MVTools2, basic)

    DENOISE — Dust removal (RemoveDirt)
    -----------------------------------------------------------------------------------------
    Temporal detection and removal of dust and point defects.

    Parameter              Range                        Default        Description
    denoise_mode           removedirtmc/removedirt      removedirtmc   Algorithm
    denoise_strength       1-24                         10             Detection threshold
    denoise_dist           1-20                         3              Spatial radius
    denoise_grey           true/false                   false          Luma only

    Note: removedirtmc (with motion compensation) is recommended for better results.
           The removedirt mode is faster but less accurate.


=============================================================================================
                         9.  ENHANCEMENT FILTERS
=============================================================================================

    LUMA LEVELS — Brightness levels
    -----------------------------------------------------------------------------------------
    Global adjustment of brightness, contrast, saturation and hue.

    Parameter       Range              Default     Description
    Lum_Bright      -255 to 255        0           Brightness offset
    Lum_Contrast    0.1 to 3.0         1.05        Contrast multiplier
    Lum_Sat         0.1 to 3.0         1.10        Colour saturation
    Lum_Hue         -180 to 180        0           Hue rotation (degrees)
    Lum_GammaY      0.1 to 3.0         1.30        Gamma curve

    Note: GammaY > 1 brightens midtones, < 1 darkens them.
    This is often the most useful parameter for underexposed old films.

    GAMMAC — Advanced colour correction
    -----------------------------------------------------------------------------------------
    Automatic correction of colour drift between RGB channels.
    Particularly useful for films exhibiting yellowing, blue or red colour casts.

    Parameter       Range              Default     Description
    LockChan        -3 to 2            1           Reference channel
    LockVal         1-255              250         Lock threshold
    Scale           0-2                2           Scale mode
    Th              0.0-1.0            0           Low threshold
    HiTh            0.0-1.0            1           High threshold
    X               0-2000             0           X offset of analysis area
    Y               0-2000             0           Y offset of analysis area
    W               0-4000             0           Area width (0 = all)
    H               0-4000             0           Area height (0 = all)
    Omin            0-255              0           Minimum output
    Omax            0-255              255         Maximum output
    Show            true/false         false       Show visualisation
    Verbosity       0-6                0           Log detail level

    LockChan: 0=Red, 1=Green, 2=Blue, -1=target value, -2=average, -3=median.
    Click the (i) button to open the detailed GamMac documentation.

    SHARPEN — Sharpening
    -----------------------------------------------------------------------------------------
    Image sharpening. Always applied last in the pipeline
    to avoid amplifying artefacts from other filters.

    Parameter              Range              Default     Description
    Sharp_Mode             simple/edge        simple      Algorithm
    Sharp_Strength         1-20               8           Sharpening strength
    Sharp_Radius           0.5-5.0            1.5         Blur radius (mask)
    Sharp_Threshold        0-100              0           Anti-noise threshold

    Notes:
        • simple: global Sharpen(), effective but may amplify residual grain.
        • edge: contours only via Sobel mask + gamma (LimitedSharpenFaster
          technique). Preserves smooth areas. Recommended after Degrain.

                        10.  CUSTOM FILTERS
=============================================================================================

AvyScanLab lets you create your own AviSynth filters and integrate them into the restoration
pipeline with a complete control interface.


    FILTER EDITING FORM
    -----------------------------------------------------------------------------------------

    The form opens by clicking the "+" button (creation) or the pencil button ✎
    (editing an existing filter). It is divided into three areas separated by a
    resizable splitter. The size, window position and area distribution
    are remembered between sessions.


    UPPER AREA — Name, plugins and scripts
    .........................................................................................

    Filter name
        Free text input field. If left empty, the default name "Custom" is assigned.

    Region Drawing (Region Draw)
        Checkbox that enables the ability to draw a region on the image for this filter.
        When checked, a drop-down menu appears to choose the region mode:

            • X / Y / W / H           : position and size of the area
            • Crop (L / T / R / B)    : cropping margins

        The corresponding placeholders (X, Y, W, H or left, top, right, bottom) are automatically detected in the code.

    Plugins (TryLoadPlugin)
        Button "+ Add DLL...": opens a file selector to add one or more AviSynth plugins (.dll).
        Each added DLL appears with its full path (green if the file exists, red otherwise) and a delete button.
        DLLs are loaded via TryLoadPlugin() in the generated script.

    Scripts (Import .avsi)
        Button "+ Add AVSI...": opens a selector to add script files (.avsi or .avs).
        Selected files are automatically copied to the application's Plugins/Scripts/ folder.
        Only the file name is stored.
        When generating the AviSynth script, AvyScanLab automatically inserts an Import() instruction for each referenced script:
            • TryLoadPlugin("C:/.../MyPlugin.dll")
            • Import("C:/.../Plugins/Scripts/MyFunctions.avsi")


    CENTRAL AREA — AVISYNTH CODE EDITOR
    .........................................................................................

    Write AviSynth code in the editor. Use "c" as the clip variable and curly braces {name} to define
    parameters adjustable from the interface:

        c = MyFilter(c, strength={sigma}, mode={bt_mode})

    Placeholders {name} are detected automatically in real time.
    Each new placeholder generates a control line in the lower area.
    If a placeholder is removed from the code, its control disappears as well.

    Converting a variable to a parameter
        • Select a line of the type "variable = value" with the mouse in the editor.
        • AvyScanLab offers to automatically convert it to a parameter: the value is replaced by {variable}
          in the code and a new control is created with the original value as the default.
          This assistance can be disabled for the current session.

    Button ▶ Preview
        Tests the filter in real time in the player without leaving the form.
        The filter is temporarily activated for preview.


    LOWER AREA — Detected parameters
    .........................................................................................

    Each placeholder {name} detected in the code appears as a horizontal row containing the following elements:

        👁  Eye button
                Highlights the placeholder in the code editor (toggles green when active).

        ✕   Delete button
                Removes the control and replaces the placeholder in the code with its default value.
                Confirmation is requested.

        {name}
                Placeholder name displayed in blue, monospace font.

        Type
                Drop-down list to choose the control type:

            slider
                        Slider with Min, Max, Step and Default (Dflt) fields.
                        Values are automatically snapped to the step.
                        With a step of 2, only even values are allowed.
                        Button ½: if enabled, the parameter value is divided by 2 in half-resolution mode (preview),
                        which is useful for parameters related to pixel size.

            combo (drop-down list)
                        "Values" field: enter values separated by commas (e.g.: val1, val2, val3).
                        "Dflt" field: default value (the first option is used if the field is empty).

            checkbox
                        "On" and "Off" fields: values associated with each state (default "true" and "false").
                        "Dflt" field: default value.

            text
                        Free text input field. "Dflt" field: default value.

        💬  Description field: optional text that will be displayed as a tooltip when the user hovers over the parameter in the filter panel.


    FORM BUTTONS
    -----------------------------------------------------------------------------------------

    Delete          Permanently deletes the filter (red button, on the left).
                    Confirmation is requested.
                    The message differs depending on whether it is a built-in filter or a user-created filter.

    ↑ Export        Saves the filter to a .json file that you can share with other users.


    ❐ Duplicate     Creates a copy of the filter with the suffix "(2)".
                    The copy receives a new identifier and is placed right after the original.
                    Useful for applying the same processing multiple times with different settings.

    ? Help          Displays contextual help about the form.

    Cancel          Closes the form without saving changes.

    Save            Saves the filter and closes the form (green button).


    USING A CUSTOM FILTER
    -----------------------------------------------------------------------------------------

    The filter appears in the left column of the filter panel with:
        • An ON/OFF button to activate/deactivate the filter
        • An arrow button ▶ to open the parameter panel
        • A pencil button ✎ to open the editing form

    Parameters are displayed in the right column of the filter panel with the
    defined controls (sliders, lists, checkboxes, text fields).


    EXPORT / IMPORT A FILTER
    -----------------------------------------------------------------------------------------

    Export: In the editing form, click "↑ Export".
               The filter is saved to a .json file that you can share with other users.

    Import: In the filter panel, click the ↓ button next to the "+".
               Select an exported .json file. Imported filters receive a new identifier to avoid duplicates.

    Notes:
            • DLL paths are absolute.
              If the filter uses an external plugin, make sure the DLL is present at the same location on the target machine, or edit the filter
              after import to correct the path.

            • When exporting a filter, only the .avsi file names are saved (not their content).
              If you share a filter with another user, also provide the required .avsi files.

    Storage location: %AppData%\AvyScanLab\custom_filters.json

    GLOBAL PRESETS (filters)
    -----------------------------------------------------------------------------------------
    The Info > Presets menu allows you to save and restore your filter configurations.

    Action              Description

    The "Default" preset is always present and appears first.

    Location: %AppData%\AvyScanLab\encoding_presets.json


=============================================================================================
                              12.  ENCODING AND RENDERING
=============================================================================================

Click the REC button (red) in the transport bar to open the encoding
panel.

    STEP 1 — Clip selection
    -----------------------------------------------------------------------------------------
    Check the clips to encode in the list. Use "Select all" to encode them all at once (batch).
    Each clip can have a different output name and encoding preset.
    Each preset can include a different destination folder, making it easy to organise renders of multiple clips.

    STEP 2 — Destination folder
    -----------------------------------------------------------------------------------------
    Click the "Browse" button or enter the path.
    Available disk space is displayed in real time.

    STEP 3 — Encoder selection
    -----------------------------------------------------------------------------------------
    Encoder        Type             Usage
    x264           Lossy            Distribution (H.264)
    x265           Lossy            Distribution (HEVC)
    FFV1           Lossless         Professional archiving
    UT Video       Lossless         Fast editing
    ProRes         Lossy HQ         Post-production
    TIFF seq.      Lossless         Individual images (archiving)
    PNG seq.       Lossless         Individual images
    JPG seq.       Lossy            Individual images

    STEP 4 — Container
    -----------------------------------------------------------------------------------------
    MKV, MP4, AVI or MOV (for video codecs). Ignored for image sequences.

    STEP 5 — Advanced settings (lossy codecs only)
    -----------------------------------------------------------------------------------------
    Quality mode:
        CRF      Constant quality (default). The lower the value, the better the quality.
                 Typical values: 18-23. The slider goes from 0 to 51.
        Bitrate  Fixed bitrate in Mb/s. The recommended minimum is automatically calculated based on the resolution
                 and chosen chroma.

    Chroma subsampling:
        4:2:0    Standard (smaller file)
        4:2:2    Better colour quality
        4:4:4    Maximum quality (large file)

        Note: bitrate and chroma are linked.
               Changing chroma adjusts the recommended minimum bitrate.
               Entering a high bitrate may automatically adjust chroma to the maximum supported level.

    Output resolution:
        Original   Source resolution preserved
        1080p      1920x1080
        720p       1280x720
        576p       720x576 (PAL)
        480p       720x480 (NTSC)

        Note: odd dimensions are automatically adjusted to even numbers (x264/x265 requirement) via a padding filter.

    DURING ENCODING
    -----------------------------------------------------------------------------------------
    • Video playback is automatically stopped
    • The interface is locked (filters, transport, menus, clip tabs)
    • Only the "Stop" button remains active
    • A progress bar displays the completion percentage
    • Closing the application automatically stops the current encoding

    "Shut down after encoding" option: check this box to automatically shut down the computer after a successful batch encoding.

    Technical note: encoding uses a dedicated render script (ScriptRender.avs) where preview mode is disabled.


=============================================================================================
                             13.  RECOMMENDED WORKFLOW
=============================================================================================

    1. Open your source (drag and drop or source field).

    2. Navigate to a representative frame (neither too dark nor too bright).

    3. Activate filters one by one in pipeline order:

       a) Crop       — Remove black borders or adjust framing.

       b) Denoise    — Remove dust and scratches. The "standard" preset works in most cases.
                       Increase strength if defects persist.

       c) Degrain    — Start with the "Medium grain" preset, then increase thSAD if grain persists.
                       Warning: values that are too high produce a "plastic" effect.

       d) Luma       — Adjust brightness and contrast.
                       GammaY is often the most useful parameter for brightening old films.

       e) GamMac     — Correct colour drift (yellowing, blue cast, red cast, etc.).

       f) Sharpen    — Sharpen as the final step. The "edge" mode better preserves smooth areas
                       and avoids amplifying residual grain.

    4. Check the result by reviewing several passages of the film.

    5. Save your configuration as a preset via the 👁 button to easily reuse it on other clips
       from the same film or similar films.

    6. If you have multiple clips: load them all, then apply the appropriate presets.

    7. Open the Record panel, choose encoding parameters.
       Each film can have a different encoding preset, making it easy to organise renders.
       Each film can or cannot be included in an encoding batch.
       Start the render!

    Tip: activate half-resolution mode (1/2) while adjusting parameters to speed up preview,
              then deactivate it to check the result at full resolution before encoding.

                           14.  TIPS AND SHORTCUTS
=============================================================================================

    • The mouse scroll wheel on numeric fields adjusts the value by step.
    • Half-resolution mode (1/2 button) considerably speeds up preview.
    • All parameters are automatically saved to ScriptUser.avs on each change.

    • The Record panel is restored on startup if it was open when the application was closed.
    • Tooltips (on hover) on each parameter display ranges and tips.
    • Dragging and dropping multiple files automatically creates one clip per file.

    • The guided tour (menu) lets you discover the interface basics step by step.

    • The "Copy" button in the error window copies the message and details.
